# Gobernanza de VibeSpec

Esta sección define cómo un equipo u organización **adopta, aplica y hace
evolucionar** el estándar VibeSpec. Responde a tres preguntas clave:

1. **¿Cómo empezamos?** → `adopcion.md`
2. **¿Qué hacemos si no podemos cumplir algo?** → `desviaciones.md`
3. **¿Cómo mejoramos el estándar con el tiempo?** → `evolucion.md`

## Principio de autoridad

El estándar VibeSpec no tiene un dueño único; se gobierna mediante
**responsabilidad distribuida**:

- **Cada desarrollador** es responsable de aplicar las checklists en su
  trabajo diario y de documentar sus desviaciones.
- **Los revisores de PR** verifican el cumplimiento y la trazabilidad.
- **El líder técnico (o arquitecto)** ratifica las desviaciones que
  afectan a nivel Pro y coordina las revisiones trimestrales del
  estándar.
